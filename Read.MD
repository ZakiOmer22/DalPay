# рҹ”җ AUTH ARCHITECTURE

### Flow:

```
Login request (Next.js)
   вҶ“
POST /auth/login (Express)
   вҶ“
JWT generated (access + refresh)
   вҶ“
Stored in:
   - HTTP-only cookie (refresh)
   - memory/localStorage (access)
```

### Middleware:

```
auth.middleware.ts
   в”ңв”Җв”Җ verify token
   в”ңв”Җв”Җ attach req.user
   в””в”Җв”Җ block unauthorized routes
```

# рҹ§© WHY THIS STRUCTURE IS CORRECT

### 1. Separation of concerns

* Frontend = UI only
* Backend = logic only
* DB = storage only

### 2. Scalable

* Can split into microservices later
* Can add mobile app easily

### 3. Clean deployment

* Vercel вҶ’ frontend
* Render вҶ’ backend
* Neon вҶ’ DB

---

# вҡ пёҸ COMMON MISTAKES YOU AVOID NOW

вқҢ Mixing API routes inside Next.js (bad for scale)
вқҢ Direct DB access from frontend
вқҢ No shared types (causes bugs)
вқҢ No service layer in backend
вқҢ Flat structure (hard to maintain)
